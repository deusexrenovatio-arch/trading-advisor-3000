## Phase

- Name: P1 - Full Dependency and Reference Inventory
- Status: completed

## Objective

- Produce a machine-readable full map of legacy path references, with risk labels and wave ownership so no high-risk rename dependency is left unassigned.

## In Scope

- Inventory generation logic for legacy namespace references.
- JSON and markdown inventory artifacts with grouped references.
- Risk classification and wave assignment for every active reference cluster.
- Explicit exclusion policy for historical frozen zones.

## Out Of Scope

- Physical renaming of docs, runtime paths, or tests.
- CI selector and CODEOWNERS cutover.
- Compatibility bridge implementation.

## Change Surfaces

- GOV-RUNTIME
- ARCH-DOCS
- PROCESS-STATE

## Constraints

- Inventory must be reproducible from tracked files, not manually assembled.
- Risk labels must separate low docs links from medium script/test paths and high runtime or selector dependencies.
- Historical frozen artifacts stay reported but excluded from active workload.

## Acceptance Gate

- Machine-readable inventory artifact exists and is generated by script.
- Every active high-risk reference is assigned to an explicit future wave.
- No legacy reference class remains unlabeled in the active scope.

## Disprover

- Introduce a new legacy reference in a tracked script file and verify inventory output reflects it with risk and wave labels.
