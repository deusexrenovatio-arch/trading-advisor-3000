## Expansion Criteria (Fail-Closed)

- Expansion to `live transport closure` and `durable storage closure` is blocked unless all checks below pass together:
  - `critical tasks with explicit solution class >= 4` in the active pilot window.
  - `blocked shortcut claims <= 1` in the same window.
  - `staged-vs-target declarations` include at least one `staged` and at least one `target` declaration.
  - every blocked shortcut claim has an explicit operator note confirming it was a true-positive block, not unresolved validator noise.
- If any criterion is missing, ambiguous, or unresolved, contour expansion stays blocked and pilot scope remains unchanged.
